# בוט טלגרם לעסקים ישראליים

הקימו בוט טלגרם לעסק הישראלי שלכם. המדריך מכסה יצירת בוט, מענה אוטומטי בעברית, קביעת תורים, ניהול הזמנות, שעות פעילות (ראשון-חמישי), קישורי תשלום, והתראות ללקוחות. מיועד לבעלי עסקים בלי רקע טכני.

## בעיה

בעלי עסקים קטנים בישראל מפסידים לקוחות כל יום כי הם לא מספיקים לענות. הודעות מגיעות בשבת, בלילה, או כשהבעלים באמצע טיפול עם לקוח אחר. התוצאה: שאלות בלי מענה, תורים שהוחמצו, והכנסות שאבדו.

רוב העסקים הישראליים משתמשים בוואטסאפ, אבל טלגרם מציע אוטומציה חזקה שוואטסאפ לא מאפשר: מקלדות אינליין, תשובות אוטומטיות, תפריטים מסודרים ואינטגרציית תשלומים, והכל בלי לכתוב שורת קוד אחת. הבעיה היא שרוב בעלי העסקים לא יודעים איך להקים בוט טלגרם, והמדריכים הקיימים מיועדים למתכנתים.

הסקיל הזה סוגר את הפער. הוא מלווה בעלי עסקים צעד אחרי צעד ביצירת בוט טלגרם מקצועי שמטפל בחלקים החוזרים בתקשורת עם לקוחות: מענה לשאלות נפוצות, קביעת תורים, קבלת הזמנות ושליחת עדכונים, בעברית, עם שעות פעילות ותרבות ישראלית מובנית.

## מתי להשתמש בסקיל הזה

- בעל עסק רוצה לייצר מענה אוטומטי ללקוחות בטלגרם
- מישהו שואל איך ליצור בוט טלגרם לעסק הישראלי שלו
- פרילנסר רוצה קביעת תורים דרך טלגרם
- מסעדה או בית קפה רוצים לקבל הזמנות דרך בוט
- נותן שירות צריך מענה אוטומטי מחוץ לשעות הפעילות בעברית
- מישהו רוצה לשלוח עדכונים ללקוחות דרך ערוץ טלגרם

## מתי לא להשתמש

- **אוטומציה לוואטסאפ** -- השתמשו ב-`israeli-whatsapp-business`
- **בניית בוט בקוד** (Node.js, Python) -- השתמשו ב-`telegram-bot-builder`
- **בוטים קוליים או IVR** -- השתמשו ב-`hebrew-voice-bot-builder`
- **ניתוב פניות שירות/הלפדסק** -- השתמשו ב-`israeli-customer-support-automator`
- **צ'אטבוט שיווקי לליד ג'נרציה** -- השתמשו ב-`hebrew-chatbot-builder`

## שלב 1: יצירת הבוט עם BotFather

הכלי BotFather הוא הכלי הרשמי של טלגרם ליצירת בוטים. לא צריך לדעת לתכנת.

### יצירת בוט צעד אחרי צעד

1. **תפתחו טלגרם** ותחפשו `@BotFather`
2. תתחילו שיחה ותשלחו `/newbot`
3. **תבחרו שם תצוגה**, זה מה שהלקוחות רואים. תשתמשו בשם העסק:
   - טוב: `מסעדת שמש` או `Shemesh Restaurant`
   - טוב: `ספרא - תורים`
4. **תבחרו שם משתמש**, חייב להסתיים ב-`bot` ולהיות ייחודי:
   - טוב: `shemesh_restaurant_bot`, `sapra_booking_bot`
   - לא טוב: `mybot` (גנרי מדי, כנראה תפוס)
5. **תשמרו את הטוקן** ש-BotFather שולח. הוא נראה ככה: `7123456789:AAHxxx...`. תשמרו אותו במקום בטוח, הוא שולט בבוט שלכם.

### הגדרת פרופיל הבוט

תשלחו את הפקודות האלה ל-@BotFather:

```
/setdescription
```
תכתוב תיאור בעברית שלקוחות רואים לפני שמתחילים שיחה:
> ברוכים הבאים! אני הבוט של [שם העסק]. אפשר לקבוע תור, לראות תפריט, ולקבל מידע על שעות פעילות. זמין 24/7!

```
/setabouttext
```
טקסט קצר ל"אודות" של הבוט:
> בוט שירות לקוחות של [שם העסק] - תורים, הזמנות, ומידע

```
/setuserpic
```
תעלו את הלוגו של העסק או תמונה מקצועית.

```
/setcommands
```
תגדירו את תפריט הפקודות:
```
start - התחל שיחה
hours - שעות פעילות
book - קבע תור
menu - תפריט / שירותים
contact - צור קשר
help - עזרה
```

## שלב 2: הודעת פתיחה וברכה אוטומטית

כשלקוח פותח את הבוט בפעם הראשונה ולוחץ "Start", הוא צריך לקבל ברכה חמה בעברית עם אפשרויות ניווט ברורות.

### תבנית הודעת פתיחה

```
שלום! 👋
ברוכים הבאים ל[שם העסק].

איך אפשר לעזור?
```

תצרפו **מקלדת אינליין** עם האפשרויות העיקריות:

| שורה | טקסט כפתור | פעולה |
|------|------------|-------|
| שורה 1 | 📅 קביעת תור | -> מסלול הזמנת תור |
| שורה 1 | 📋 תפריט/שירותים | -> רשימת שירותים |
| שורה 2 | 🕐 שעות פעילות | -> שעות פעילות |
| שורה 2 | 📍 מיקום | -> קישור למפה |
| שורה 3 | 📞 דברו איתנו | -> פרטי קשר |

### טיפים חשובים

- תשמרו על ברכה קצרה, לא יותר מ-3 שורות לפני הכפתורים
- עברית כשפת ברירת מחדל
- תוסיפו כפתור "English" אם אתם משרתים תיירים או דוברי אנגלית
- הברכה צריכה להרגיש אישית, לא תאגידית. "שלום!" עדיף על "שלום וברוכים הבאים לשירות הלקוחות שלנו"

## שלב 3: מענה אוטומטי לשאלות נפוצות

תגדירו תשובות אוטומטיות לשאלות שמקבלים 10 פעמים ביום.

### קטגוריות FAQ נפוצות לעסקים ישראליים

**שעות פעילות:**
```
🕐 שעות פעילות:

ראשון - חמישי: 09:00 - 18:00
שישי: 09:00 - 14:00
שבת: סגור

חגים: לפי עדכונים בערוץ שלנו
```

**מיקום:**
```
📍 המיקום שלנו:

[כתובת מלאה]
[עיר]

🗺 לניווט: [קישור ל-Google Maps]
🅿️ חניה: [פרטי חניה]
```

**מחירים / תפריט:**
```
📋 התפריט שלנו:

[תציגו כקטגוריות במקלדת אינליין או תשלחו PDF/תמונה]
```

**יצירת קשר:**
```
📞 צרו קשר:

טלפון: 050-XXX-XXXX
ווטסאפ: [קישור לוואטסאפ]
אימייל: info@business.co.il

או תכתוב לנו כאן ונחזור אליכם בשעות הפעילות!
```

### FAQ רב-שכבתי עם מקלדת אינליין

לעסקים עם הרבה שאלות, תשתמשו במבנה עץ:

```
תפריט ראשי
├── 📋 שירותים
│   ├── תספורת גברים - ₪80
│   ├── תספורת נשים - ₪120-200
│   ├── צבע - ₪250+
│   └── ⬅️ חזרה לתפריט
├── 💰 מחירים
│   ├── מחירון מלא (PDF)
│   └── ⬅️ חזרה לתפריט
├── 📅 קביעת תור
└── 🕐 שעות פעילות
```

## שלב 4: לוגיקת שעות פעילות

שעות הפעילות בישראל שונות מהעולם. הבוט חייב לטפל בזה נכון.

### לוח הזמנים הישראלי

| יום | סימון | שעות טיפוסיות | סטטוס |
|-----|-------|---------------|-------|
| ראשון | א׳ | 08:00/09:00 - 17:00/18:00 | פתוח |
| שני | ב׳ | 08:00/09:00 - 17:00/18:00 | פתוח |
| שלישי | ג׳ | 08:00/09:00 - 17:00/18:00 | פתוח |
| רביעי | ד׳ | 08:00/09:00 - 17:00/18:00 | פתוח |
| חמישי | ה׳ | 08:00/09:00 - 17:00/18:00 | פתוח |
| שישי | ו׳ | 08:00/09:00 - 13:00/14:00 | יום קצר |
| שבת | ש׳ | סגור | שבת |

### מענה אוטומטי מחוץ לשעות

כשמישהו שולח הודעה מחוץ לשעות הפעילות, תענו אוטומטית:

**ערב/לילה רגיל (ראשון-חמישי):**
```
היי! 🌙
קיבלנו את ההודעה שלך.
אנחנו פעילים ראשון-חמישי בין 09:00-18:00.
נחזור אליך מחר בבוקר!

בינתיים אפשר:
📅 לקבוע תור
📋 לראות תפריט
🕐 לבדוק שעות פעילות
```

**שישי אחה"צ / שבת:**
```
שבת שלום! 🕯
קיבלנו את ההודעה. נחזור אליך ביום ראשון.

בינתיים אפשר לקבוע תור או לראות מידע דרך התפריט.
```

**ערב חג:**
```
חג שמח! 🎉
אנחנו בחופשת חג ונחזור ב[תאריך חזרה].

לתורים ומידע, השתמשו בתפריט למטה.
```

### מודעות לחגים

חגים ישראליים עיקריים שבהם עסקים סגורים:
- **ראש השנה** -- יומיים
- **יום כיפור** -- יום אחד (כמעט הכל סגור)
- **סוכות** -- יום ראשון ואחרון
- **פסח** -- יום ראשון ואחרון
- **שבועות** -- יום אחד
- **יום העצמאות** -- יום אחד

הגדירו את הבוט לבדוק לוח חגים ולהפעיל מצב "מענה חג" אוטומטי. רוב הפלטפורמות ללא קוד תומכות בלוגיקה מבוססת תאריכים או שינוי הודעות מתוזמן.

### הודעת אישור

אחרי קביעת תור:
```
✅ התור נקבע!

שירות: תספורת
תאריך: יום שני, 3 באפריל
שעה: 10:00
כתובת: [כתובת העסק]

נשלח תזכורת יום לפני.
לביטול או שינוי, שלחו /cancel
```

### תזכורות

- **24 שעות לפני**: "תזכורת: יש לך תור מחר ב-10:00 ב[שם העסק]. לאישור לחצו ✅"
- **שעתיים לפני**: "התור שלך ב[שם העסק] בעוד שעתיים. נתראה! 📍 [קישור לניווט]"
- **אחרי אי-הגעה**: חכו 30 דקות, ואז "לא הספקת להגיע? אפשר לקבוע תור חדש כאן: /book"

### חשוב: לוגיקת תורים ליום שישי

כשלקוח מנסה לקבוע תור ליום שישי:
- הציגו רק שעות בוקר (עד 13:00 או 14:00 לפי העסק)
- לעולם אל תציגו שבת כאפשרות
- אחרי חמישי, היום הבא הזמין צריך להיות ראשון

## שלב 7: אינטגרציית תשלומים

### אפשרות 1: קישורי תשלום חיצוניים

הגישה הכי פשוטה לעסקים ישראליים. צרו קישור תשלום מספק התשלומים הקיים ושלחו אותו בבוט:

```
💳 לתשלום:

סה"כ לתשלום: ₪103

בחרו אמצעי תשלום:
[💳 כרטיס אשראי] -> קישור לדף תשלום
[📱 ביט] -> קישור תשלום ביט
[🏦 העברה בנקאית] -> פרטי חשבון
```

**שערי תשלום ישראליים פופולריים:**
- **חשבונית ירוקה (Green Invoice)** -- יוצר קישורי תשלום עם חשבונית אוטומטית
- **רווחית (Rivhit)** -- גביית תשלומים עם אינטגרציית הנהלת חשבונות
- **PayMe** -- יצירת דף תשלום פשוט, גם משמש לקבלת תשלומי ביט בפלטפורמות כמו Wix
- **iCount** -- חשבוניות עם קישורי תשלום
- **משולם (Meshulam)** -- סליקת תשלומים לעסקים קטנים
- **טרנזילה (Tranzila)** -- מסלקת תשלומים ישראלית ותיקה (משנת 1999), תומכת בש"ח, שלושה מצבי שילוב (Iframe / Hosted Fields / API v2). יש אינטגרציות קהילתיות עם טלגרם דרך Make.com.
- **ביט (Bit)** -- אפליקציית התשלום הפופולרית בישראל, בבעלות בנק הפועלים. יש פורטל מפתחים בכתובת `developer.bitpay.co.il` לאינטגרציות עסקיות. **מגבלות חשובות למוכרים**: רק ש"ח (לא מטבע זר), מקסימום ₪5,000 לעסקה אחת, מקסימום ₪20,000 בחודש למוכר, תשלום חד-פעמי בלבד (בלי תשלומים, בלי מנויים). מתאים למכירות בסכומים נמוכים (הזמנות אוכל, שירותי יופי), לא לרכישות גדולות או חיוב חוזר.
- **פייבוקס (PayBox)** -- ארנק דיגיטלי בבעלות בנק דיסקונט, עם כ-2.5 מיליון משתמשים, התמקדות בהעברות בין אנשים פרטיים ותשלומים קבוצתיים. אין API ציבורי רשמי לסליקה לעסקים נכון ל-2026; בדרך כלל עסקים חושפים שם משתמש פייבוקס להעברות ידניות במקום לייצר קישורי תשלום אוטומטיים.

### אפשרות 2: תשלומים מובנים של טלגרם (Payment API)

אפשרות ביניים בין קישורים חיצוניים ל-Telegram Stars. לטלגרם יש Payment API מובנה (מתועד ב-`core.telegram.org/bots/payments`): מחברים ספק תשלומים דרך @BotFather (בתפריט `/mybots` יש כניסה "Payments" שבה מקשרים ספק נתמך), ואז הבוט יכול לשלוח הודעות חשבונית מובנות שהלקוחות משלמים בלי לצאת מטלגרם. זה משאיר את התשלום בתוך הצ'אט במקום להקפיץ את הלקוח ללשונית דפדפן חיצונית, ועדיין מסלק כסף אמיתי (בניגוד ל-Telegram Stars שמיועד למוצרים דיגיטליים).

ש"ח הוא מטבע נתמך ב-Bot Payments API (קוד מטבע `ILS`, סכום מינימלי ₪3.68, סכום מקסימלי ₪36,788.20 לחשבונית). אבל מסלול החיבור דרך @BotFather מציע רק ספקים שלטלגרם יש איתם אינטגרציה ישירה, ונכון לאמצע 2026 הסולקים הישראליים (טרנזילה, משולם, Cardcom) לא נמצאים ברשימת הספקים הנתמכים רשמית של טלגרם. המשמעות המעשית:

- אם הקלפים של הלקוחות שלכם הם ישראליים אבל אתם בסדר עם סליקה דרך Stripe בש"ח, ה-Payment API המובנה עובד היום דרך האינטגרציה הגלובלית של Stripe.
- אם אתם חייבים סליקה דרך סולק ישראלי מסיבות מס/דיווח, שלחו קישור תשלום חיצוני (אפשרות 1) או בנו Mini App (אפשרות 4) שמטמיע את ה-iframe של הספק הישראלי שלכם. ה-Payment API המובנה של טלגרם הוא כרגע לא הכלי הנכון להגדרת מוכר ישראלי "טהורה".

תמיד בדקו את רשימת הספקים החיה בתפריט Payments של @BotFather לפני שמתחייבים לאפשרות הזו -- טלגרם מוסיף ספקים עם הזמן.

### אפשרות 3: Telegram Stars (למוצרים דיגיטליים)

טלגרם תומך ב-Telegram Stars לרכישות בתוך האפליקציה של מוצרים ושירותים דיגיטליים. מתאים ל:
- ייעוץ דיגיטלי
- קורסים אונליין
- קבצים דיגיטליים / תבניות
- גישה לתוכן פרימיום

שימו לב: Telegram Stars מיועד למוצרים דיגיטליים בלבד. מוצרים פיזיים ושירותים צריכים להשתמש בקישורי תשלום חיצוניים או ב-Payment API המובנה.

### אפשרות 4: Telegram Mini Apps (TWA)

אפליקציית Mini App היא דף אינטרנט מלא (HTML/CSS/JS) שנפתח בתוך טלגרם, עם גישה ל-API ייחודי של טלגרם (זהות המשתמש, ערכת נושא, רטט, MainButton, חלון תשלום מובנה). מתועד ב-`core.telegram.org/bots/webapps`. עד 2026 Mini Apps הפכו לערוץ מסחר מרכזי בעולם -- טלגרם חצתה מיליארד משתמשים פעילים בחודש, וארנקים מבוססי TON בתוך הצ'אט נפוצים מאוד.

לעסק ישראלי קטן, Mini App מתאים כשהבוט צריך חוויית משתמש עשירה יותר ממקלדות אינליין: רשת קטלוג עם תמונות, בורר תאריכים/שעות לקביעת תור, טופס הזמנה רב-שלבי. הזרימה נשארת בתוך טלגרם -- הלקוח לא פותח לשונית דפדפן -- ואותו URL עובד באייפון, באנדרואיד ובדסקטופ בלי המתנה לאישור חנות אפליקציות.

מתי לא להשתמש ב-Mini App: בוט פשוט של שאלות נפוצות וקביעת תורים עם חמישה כפתורי אינליין לא צריך Mini App. הוספת אחד מכניסה קוד פרונטאנד שצריך לתחזק והוסטינג HTTPS שצריך לשלם עליו. השארו עם מקלדות אינליין עד שמסלול הלקוח דורש יותר.

ההגדרה דרך @BotFather: `/newapp` (או `/myapps` -> "Edit Bot" -> "Configure Mini App") מחבר Mini App לבוט. הדף מוגש מכתובת HTTPS שלכם; @BotFather רק רושם אותו ונותן ללקוח כפתור לפתוח אותו.

ספקי תשלום ישראליים לא צריכים אינטגרציה ייעודית עם טלגרם כדי לעבוד בתוך Mini App -- אפשר להטמיע את אותו iframe של טרנזילה או דף תשלום של PayMe שהייתם משתמשים בו באתר רגיל.

## שלב 8: התראות ושידורים ללקוחות

### פוסטים בערוץ מול שידור DM (הבחנה חשובה)

יש שני דברים שונים שאנשים קוראים להם "שידור":

1. **פרסום בערוץ טלגרם** -- הודעה אחת לכל המנויים שבחרו להצטרף לערוץ. הבוט מפרסם בתור מנהל של הערוץ; זה נחשב כהודעה יחידה ל-API של טלגרם בלי קשר לכמה מנויים יש, ולכן זה לא כפוף להגבלות הקצב של בוט. ערוצים הם הכלי הנכון למבצעים, עדכונים שבועיים וברכות חג לכל הקהל שלכם.
2. **שליחת DM פרטי לכל לקוח ברשימה** -- הבוט קורא ל-`sendMessage` פעם אחת לכל לקוח. זה כן כפוף להגבלות הקצב של טלגרם (ראו למטה), וגם כפוף לחוק הספאם הישראלי (סעיף 30א) כי זאת הודעה מסחרית לא מבוקשת לאדם פרטי.

בכל מקום שאפשר, העדיפו ערוץ לתוכן פרסומי. שמרו שידורי DM להודעות שירות שהלקוח ביקש (סטטוס הזמנה, תזכורת לתור), או למבצעים שנשלחים רק ללקוחות שנתנו הסכמה מפורשת.

### ערוץ טלגרם לעדכונים

צרו ערוץ טלגרם מקושר לבוט לשידור עדכונים:

1. צרו ערוץ חדש בטלגרם
2. קראו לו: `[שם העסק] - עדכונים`
3. הוסיפו את הבוט כמנהל
4. שתפו את הקישור לערוץ עם הלקוחות

### עמידה בחוק הספאם

לפני שמשדרים מבצעים או הודעות שיווקיות, העסק חייב לעמוד בחוק הספאם, סעיף 30א לחוק התקשורת (בזק ושידורים), התשמ"ב-1982, שנוסף בתיקון 40 (2008). החוק חל במפורש גם על אפליקציות מסרים מיידיים, ולכן שידור בטלגרם ללקוחות נחשב "דבר פרסומת" ברגע שהוא מקדם מוצר, שירות או הנחה.

שלוש דרישות חוקיות שכל שידור חייב לעמוד בהן:

1. **הסכמה מפורשת מראש** -- אסור לשלוח הודעה פרסומית ללקוח שלא נתן הסכמה אקטיבית. לקוח שפתח שיחה עם הבוט לא נתן בכך הסכמה לקבל דיוור שיווקי. תוסיפו שלב הצטרפות ברור (למשל כפתור במקלדת אינליין "כן, שלחו לי מבצעים ועדכונים") ותשמרו מי הסכים. הודעות שירות (סטטוס הזמנה, תזכורת לתור שהלקוח ביקש) אינן "דבר פרסומת" ואינן דורשות הסכמה כזו.
2. **אפשרות הסרה בכל הודעה** -- כל שידור פרסומי חייב לכלול דרך פשוטה להפסיק לקבל אותם, למשל כפתור "הסר אותי מרשימת התפוצה" או ההוראה "כדי להפסיק לקבל מבצעים, שלחו /stop". תכבדו בקשות הסרה מיד.
3. **זיהוי השולח** -- ההודעה חייבת לזהות בבירור את העסק ששולח אותה (שם העסק) ולכלול פרטי קשר. אל תשלחו מבצעים אנונימיים.

שליחת הודעות פרסומיות בלי הסכמה חושפת את העסק לפיצוי סטטוטורי של עד ₪1,000 לכל הודעה לפי סעיף 30א(י), בלי שהנמען צריך להוכיח נזק כלשהו. סעיף 30א נמצא גם בתוספת השנייה לחוק תובענות ייצוגיות, כך שדיוור המוני אחד יכול להפעיל תביעה ייצוגית שמכסה את כל הנמענים. בנוסף לחשיפה האזרחית, שליחת הודעות פרסומיות ביודעין בלי הסכמה היא עבירה פלילית; תקרת הקנס לתאגיד נכון ל-2026 היא כ-₪226,000 (מתעדכן שנתית). במקרה של ספק, תתייחסו להודעה כאל פרסומת ותקבלו הסכמה קודם. לבירור החובות המדויקות והעדכניות, היוועצו במקורות משפטיים רשמיים ב-`kolzchut.org.il` או בעורך דין.

## שלב 9: פלטפורמות ללא קוד

לא צריך לכתוב קוד בשביל לבנות בוט טלגרם עסקי. הפלטפורמות האלה מציעות עורכים ויזואליים.

### פלטפורמות מומלצות

**BotPress**
- בונה שיחות ויזואלי עם תמיכה בעברית
- תוכנית חינמית זמינה
- אינטגרציית טלגרם טובה
- עיצוב שיחות בגרירה ושחרור
- הכי מתאים ל: מסלולי שיחה מורכבים, הזמנות רב-שלביות

**n8n (אירוח עצמי או ענן)**
- אוטומציית תהליכים ויזואלית
- Telegram trigger node מובנה
- Code node ללוגיקה מותאמת (אופציונלי)
- Schedule Trigger node להודעות מתוזמנות (תזכורות, מבצעים)
- הכי מתאים ל: חיבור טלגרם לכלים עסקיים (Google Sheets, יומנים, CRM)
- אפשר לחבר לפלטפורמות ישראליות כמו Monday.com וחשבונית ירוקה

**ManyChat**
- תמיכה מלאה ומובנית בטלגרם עם בונה תהליכים ויזואלי
- רצפי אוטומציה בגרירה ושחרור
- ניהול מנויים ושליחת הודעות המוניות
- תומך גם באינסטגרם, וואטסאפ, פייסבוק מסנג'ר
- הכי מתאים ל: עסקים שכבר משתמשים ב-ManyChat לערוצים אחרים, עיצוב תהליכים ויזואלי

### השוואת פלטפורמות

| פיצ'ר | BotPress | n8n | ManyChat |
|-------|----------|-----|----------|
| תמיכה בעברית | כן | כן | חלקית |
| בונה ויזואלי | כן | כן | כן |
| תוכנית חינמית | כן | כן (אירוח עצמי) | מוגבלת |
| אינטגרציית טלגרם | מובנית | node מובנה | מובנית |
| קביעת תורים | עם מסלולים | עם אינטגרציית יומן | עם תהליכים |
| קישורי תשלום | דרך הודעות | דרך HTTP nodes | דרך הודעות |
| עקומת למידה | בינונית | בינונית | נמוכה |
| אינטגרציות ישראליות | דרך API | nodes מובנים לרבים | מוגבלות |

### Webhook מול Polling (הערת פריסה)

טלגרם מעביר עדכונים לבוט באחת משתי דרכים: **polling** (הבוט שואל שוב ושוב את טלגרם "יש הודעות חדשות?") או **webhook** (טלגרם דוחף כל עדכון לכתובת HTTPS שהבוט חושף). שיטת ה-webhook מהירה וזולה יותר אבל דורשת כתובת HTTPS ציבורית, ושיטת ה-polling לא דורשת כתובת ציבורית אבל דורשת תהליך שרץ ברציפות. עם הפלטפורמות ללא קוד שלמעלה (BotPress, n8n, ManyChat) הבחירה הזו מופשטת ונסתרת. הפלטפורמה מארחת את החיבור בשבילכם, כך שבעל עסק לא טכני אף פעם לא מגדיר webhook או לולאת polling ישירות. ההבחנה רלוונטית רק אם בהמשך עוברים לאירוח עצמי או מנסים לאבחן תקלה של "הבוט לא מגיב".

- **קצר** -- ישראלים מצפים לתשובות מהירות, לא לפסקאות

## מלכודות נפוצות

טעויות שכיחות של סוכני AI כשעוזרים להקים בוטים עסקיים ישראליים:

1. **שוכחים טיפול בשבת** -- הטעות הכי נפוצה. הבוט חייב לכלול מענה אוטומטי לשבת. לעולם אל תציגו שבת כיום זמין לתורים. שישי חייב להיות עם שעות מקוצרות. אם מדלגים על זה, בעל העסק יקבל תלונות מלקוחות דתיים ובלבול מכולם.

2. **מתייחסים לראשון כיום חופש** -- שבוע העבודה בישראל הוא ראשון-חמישי, לא שני-שישי. ראשון הוא יום עבודה רגיל. אל תעתיקו תבניות שעות פעילות מערביות.

3. **כותבים עברית מעושה/רשמית** -- אל תשתמשו ב"ניתן" (רשמי), השתמשו ב"אפשר" (יומיומי). אל תשתמשו ב"הנכם מוזמנים", השתמשו ב"בואו" או "אפשר". הבוט צריך להישמע כמו בעל חנות ידידותי, לא כמו טופס ממשלתי.

4. **מסבכים את הגרסה הראשונה** -- התחילו עם: הודעת פתיחה, שעות פעילות, שאלות נפוצות, ומסלול הזמנה או תור אחד. אל תנסו לבנות מערכת CRM שלמה ביום הראשון. תוסיפו פיצ'רים אחרי שהבסיס עובד.

5. **מתעלמים מאבטחת הטוקן** -- הטוקן של הבוט הוא כמו סיסמה. אל תשתפו אותו בצ'אטים קבוצתיים, אל תדביקו אותו בפורומים פומביים, ואל תכללו אותו בצילומי מסך. אם דלף, כל אחד יכול לשלוט בבוט. השתמשו בפקודה `/revoke` של BotFather אם הטוקן נחשף.

6. **לא מגדירים העברה לבן אדם** -- בוטים לא יכולים לטפל בהכל. כללו אפשרות "דברו עם נציג" שמעבירה את ההודעה לטלגרם האישי של בעל העסק או שולחת התראה.

7. **פורמט טלפון שגוי** -- מספרי טלפון ישראליים צריכים להיות בפורמט `050-XXX-XXXX` או `+972-50-XXX-XXXX` לבינלאומי. אל תשתמשו ב-`(050) XXX-XXXX` או פורמטים לא ישראליים אחרים.

8. **חסר כפתור "חזרה"** -- כל תת-תפריט חייב לכלול כפתור "⬅️ חזרה". לקוחות מתוסכלים כשהם תקועים בתפריט בלי דרך לחזור למסך הראשי.

9. **לא בודקים במובייל** -- רוב משתמשי הטלגרם בישראל במובייל. בדקו כל מסלול בטלפון, לא רק בדסקטופ. כפתורים שנראים טוב בדסקטופ עלולים להיות קטנים או רחבים מדי בנייד.

10. **מקודדים תאריכי חגים** -- החגים הישראליים נופלים על תאריכים שונים כל שנה (לוח עברי). השתמשו ב-API של חגים או עדכנו ידנית לפני כל עונת חגים במקום לקודד תאריכים קשיחים.

## דוגמאות

### דוגמה 1: בוט למסעדה

**תרחיש:** "קפה שמש" בתל אביב רוצים בוט טלגרם לצפייה בתפריט, הזמנת טייקאווי, ושמירת שולחנות.

**הסוכן צריך לייצר:**

הודעת פתיחה עם מקלדת אינליין:
```
שלום! ☀️
ברוכים הבאים לקפה שמש.

מה בא לכם?
[📋 תפריט] [🛒 הזמנה לטייקאווי]
[📅 הזמנת שולחן] [🕐 שעות פעילות]
[📍 איך מגיעים] [📞 צרו קשר]
```

מסלול תפריט עם קטגוריות:
```
📋 התפריט שלנו:

[☕ קפה ושתייה]
[🥐 ארוחות בוקר]
[🥗 ארוחות צהריים]
[🍰 קינוחים]
[⬅️ חזרה לתפריט הראשי]
```

שעות פעילות עם שישי/שבת:
```
🕐 שעות פעילות:

ראשון - חמישי: 07:30 - 23:00
שישי: 07:30 - 15:00
שבת: סגור

📍 רחוב דיזנגוף 99, תל אביב
```

### דוגמה 2: בוט למספרה

**תרחיש:** "סלון דנה" בחיפה רוצים קביעת תורים, מחירון, ותמונות לפני/אחרי.

**הסוכן צריך לייצר:**

מסלול הזמנה עם בחירת שירות:
```
💇 קביעת תור בסלון דנה

בחרו שירות:
[💇‍♀️ תספורת נשים - מ-₪120]
[💇‍♂️ תספורת גברים - ₪80]
[🎨 צבע - מ-₪250]
[💅 מניקור - ₪90]
[💆‍♀️ טיפול פנים - ₪180]
[⬅️ חזרה]
```

בחירת מעצב/ת (אם רלוונטי):
```
עם מי תרצו להזמין?

[דנה - בעלים ומעצבת ראשית]
[שירה - צבע ותספורות]
[יובל - תספורות גברים]
[⬅️ חזרה]
```

הודעת אישור תור:
```
✅ התור נקבע!

שירות: תספורת נשים
מעצבת: דנה
תאריך: יום שלישי, 5 באפריל
שעה: 14:00

📍 סלון דנה, רחוב הרצל 15, חיפה
🅿️ חניה חינם ברחוב

נשלח תזכורת מחר.
לביטול: /cancel
```

### דוגמה 3: בוט לרואה חשבון פרילנסר

**תרחיש:** "משה לוי, רו"ח" רוצה בוט לתיאום פגישות ייעוץ, שיתוף רשימות מסמכים, ומענה לשאלות מס נפוצות.

**הסוכן צריך לייצר:**

הודעת פתיחה בטון מקצועי:
```
שלום! 👋
משרד רו"ח משה לוי

איך אפשר לעזור?
[📅 קביעת פגישת ייעוץ]
[📄 מסמכים נדרשים]
[❓ שאלות נפוצות]
[💰 מחירון שירותים]
[📞 צרו קשר]
```

שאלות נפוצות בנושא מיסים:
```
❓ שאלות נפוצות:

[📊 מתי מגישים דוח שנתי?]
[🧾 איזה קבלות לשמור?]
[💼 פתיחת תיק עוסק מורשה]
```

```
✅ אישור ביטוח לאומי

💡 טיפ: צלמו הכל ושלחו לנו כאן, ונבדוק שיש את הכל.

[📅 קבע פגישה] [⬅️ חזרה]
```

## פתרון בעיות

### הבוט לא מגיב

- **בדקו את הטוקן**: ודאו שהעתקתם את הטוקן המלא מ-BotFather בלי רווחים מיותרים
- **הבוט לא מחובר לפלטפורמה**: ודאו שהטוקן הוזן נכון בפלטפורמה ללא קוד (BotPress, n8n וכו')
- **הפלטפורמה לא זמינה**: בדקו את דף הסטטוס של הפלטפורמה
- **בעיות Webhook**: אם משתמשים ב-webhooks, ודאו שה-URL הוא HTTPS ונגיש מבחוץ

### הודעות לא מגיעות

- **מצב פרטיות**: כברירת מחדל, בוטים בקבוצות רואים רק הודעות שמתחילות ב-`/`. השתמשו ב-`/setprivacy` של BotFather לשנות את זה אם צריך
- **הבוט חסום**: המשתמש אולי חסם את הבוט. אי אפשר לשלוח הודעות למשתמשים שחסמו אתכם
- **הגבלות קצב**: טלגרם אוכפת שלוש הגבלות נפרדות לשליחה מבוט, כולן לפי ה-FAQ הרשמי ב-`core.telegram.org/bots/faq`:
  - **לכל צ'אט**: לא יותר מהודעה אחת בשנייה לאותו צ'אט.
  - **לכל קבוצה**: לא יותר מ-20 הודעות בדקה לאותה קבוצה.
  - **שידור גלובלי**: בוט יכול לשדר לכ-30 משתמשים שונים בשנייה (תוכנית חינמית). חציית כל אחת מההגבלות תחזיר HTTP 429.
  - **שידור בתשלום (Paid broadcasts)**: בוט יכול להירשם לשלם לטלגרם עבור thruput גבוה יותר (עד 1,000 הודעות בשנייה). עולה 0.1 Stars להודעה מעל 30/שנייה החינמיים, ודורש שיתרת ה-Stars של הבוט תהיה ≥100,000 ושיהיו לבוט ≥100,000 משתמשים פעילים בחודש -- מעשי רק לעסקים גדולים, לא לעסקים קטנים ישראליים טיפוסיים.
  - לשידור DM לרשימת לקוחות, הוסיפו השהיות מכוונות בין השליחות כדי להישאר מתחת להגבלות. להכרזות המוניות, פרסמו לערוץ טלגרם במקום -- פוסט בערוץ הוא קריאת API אחת ולא צורך את התקציב שלכם לשנייה.
  - מידע הגבלות הקצב הזה וסט הפקודות של BotFather המתוארים בסקיל מותאמים ל-Bot API 10.0 (שוחרר במאי 2026); ה-Bot API ממוספר ומתעדכן באופן קבוע ב-`core.telegram.org/bots/api`, לכן בדקו את ה-changelog בראש העמוד אם פיצ'ר מתנהג אחרת מהמתואר כאן.

### בעיות תצוגת טקסט בעברית

- **ודאו שהפלטפורמה שולחת UTF-8**: כל הפלטפורמות המודרניות עושות את זה, אבל תבדקו אם הטקסט מוצג משובש
- **ערבוב LTR/RTL**: אם מספרים או טקסט באנגלית מופיעים בסדר הפוך, נסו לעטוף מקטעי טקסט בעברית בנפרד
- **יישור מקלדת אינליין**: כפתורים בעברית עשויים להופיע מיושרים לשמאל בחלק מלקוחות הטלגרם. זו התנהגות של צד הלקוח ולא משהו שאפשר לשלוט בו

### תורים מציגים שעות לא נכונות

- **אזור זמן**: ודאו שהפלטפורמה מוגדרת ל-`Asia/Jerusalem` (UTC+2, או UTC+3 בשעון קיץ)
- **מעברי שעון קיץ**: ישראל מחליפה שעון במרץ ובאוקטובר. ודאו שהלוגיקה מטפלת בזה
- **שעות שישי**: בדקו שוב ששישי מציג רק שעות מקוצרות, לא זמינות ליום מלא

### קישור תשלום לא עובד

- **הקישור פג תוקף**: חלק משערי התשלום מפיגים קישורים אחרי 24-72 שעות. צרו קישורים חדשים לכל הזמנה
- **הפניה בדפדפן מובייל**: בדקו את קישור התשלום במובייל. חלק מהשערים מפנים גרוע בדפדפן הפנימי של טלגרם
- **מטבע**: ודאו שדף התשלום מציג ILS (₪), לא USD או EUR

### הבוט נפרץ

אם אתם חושדים שמישהו השיג את הטוקן:
1. פתחו את @BotFather
2. שלחו `/revoke`
3. בחרו את הבוט שלכם
4. תקבלו טוקן חדש. עדכנו אותו בפלטפורמה מיד
5. בדקו פעילות אחרונה בבוט לזיהוי הודעות חשודות

## שרתי MCP מומלצים

- **`hebcal`** -- שרת MCP ללוח השנה העברי. הסקיל הזה חוזר ואומר לבעל העסק "לבדוק לוח חגים" או "להשתמש ב-API של חגים" לתאריכי חגים ישראליים (מודעות לחגים בשלב 4, מלכודת מספר 10). שרת ה-MCP של `hebcal` חושף תאריכי חגים מהלוח העברי כדי שסוכן יוכל לדעת מתי ראש השנה, יום כיפור, סוכות, פסח, שבועות ויום העצמאות נופלים בשנה נתונה, במקום לקודד תאריכים שמתיישנים כל שנה. השתמשו בו כדי להפעיל את מצב "מענה חג" של הבוט וכדי לוודא שמסלולי הזמנה לעולם לא מציעים תור ביום חג סגור.
